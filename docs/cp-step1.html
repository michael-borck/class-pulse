ClassPulse – Step 1: Setting Up Project Structure & Dependencies

This page guides you through initializing the ClassPulse project—laying out the folder structure and managing essential dependencies—optimized for code efficiency, maintainability, and PDF export.

Note: Visual/chart libraries are not directly setup in this step, but plan their usage for later (Chart.js/ECharts). Only include Font Awesome and Google Fonts for startup.

ClassPulse Project Directory Structure

  • ~/projects/classpulse/
  • app.py – Main Flask app
  • config.py – Project config/settings
  • models.py – Database models
  • utils.py – Utility functions
  • static/ – Static assets
  • css/ – CSS files (e.g., style.css)
  • js/ – JS files (presenter.js, audience.js)
  • templates/ – HTML templates
  • layout.html (base), login.html, dashboard.html, create_question.html, present.html, join.html...
  • instance/ – Data (e.g., classpulse.sqlite3)
  • requirements.txt – Python dependencies

Step-by-Step: Initial Project Setup

  1. Create the main project folder: 
    mkdir -p ~/projects/classpulse/{static/css,static/js,templates,instance}
  2. Navigate into the directory: 
    cd ~/projects/classpulse
  3. Initialize a Python environment (recommended): 
    python3 -m venv venv
source venv/bin/activate
  4. Create empty files (app.py, config.py, models.py, utils.py, requirements.txt): 
    touch app.py config.py models.py utils.py requirements.txt
  5. Install core Python dependencies: 
    pip install Flask flask-socketio eventlet Flask-SQLAlchemy
    Add these packages to your requirements.txt: 
    Flask
flask-socketio
eventlet
Flask-SQLAlchemy
  6. (Optional) Pre-create minimal static CSS/JS & HTML template stubs 
    echo "body { @apply bg-white text-gray-900; }" > static/css/style.css
touch static/js/presenter.js static/js/audience.js
touch templates/layout.html
Once completed, your project is ready to begin implementing app logic and UI!

Why These Libraries and Structure?

  • Tailwind CSS ensures lightweight, responsive UIs with minimal custom CSS.
  • Chart.js/ECharts let you build dynamic visualizations with few lines of code for polling results and word clouds.
  • Font Awesome adds context and visual cues to interfaces without having to create custom icons.
  • Google Fonts supports clean, professional typography across browsers and print/PDF outputs.
  • Flask-SocketIO makes real-time interactivity (polls, results) straightforward with easy Python integration.
  • project/ dir structure keeps logic, templates, and static files tidy and easy to maintain or extend.
